# BaaS (Backend as a Service) の動作確認

supabaseの利用は、基本的には非推奨。

ある程度以上になるとエッジ関数で隠さないといけないので、その場合は、クライアントからのDBコントロール機能は、あまり意味がない。

そうなると、権限設定ミスでデータが消されるとかがない分、最初からcroudflare workersで作ったほうが無難そう。

標準で認証の仕組みがあるけど、実装例を見るとわかる通り、結局、エッジ関数で隠さないといけないので、かえって面倒になりやすい。

エッジ関数不要の、めちゃくちゃシンプルな仕様の場合はありかも。

エッジ関数が必要そうなら、croudflare workers。

Firebaseは、同じ理由でも、おすすめできない。

さらにNoSQLは、負荷分散が限界に達したとき以外は利用しないほうがいい。

ただ、実装が特殊になるため、特殊な実装をしたいという欲求は満たせると思います。

どちらも、かなり限定的な用途向けだけど、Firebaseは特に限定的。
